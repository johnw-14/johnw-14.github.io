From Data to DataLoaders¶

Now that we have the images, we need to organize them for training our model. The course shows you how to do this using DataBlock. We will save 20% of the images for a validation set, while the other 80% of the images will be used to train the model.

In [9]:
bears = DataBlock(
    blocks=(ImageBlock, CategoryBlock), 
    get_items=get_image_files, 
    splitter=RandomSplitter(valid_pct=0.2, seed=42),
    get_y=parent_label,
    item_tfms=Resize(128))

Now we load the bear images into the structure we specified.

In [10]:
dls = bears.dataloaders(path)

Training the Model¶

The course also spends some time on data augmentation. For training this model we use RandomResizedCrop which crops a different portion of the image with each pass. This ensures all images are cropped to the same size for passing to the model, but the random resizing also guards against missing important features of the image. The course lesson has a lot more information on this.

In [11]:
bears = bears.new(
    item_tfms=RandomResizedCrop(224, min_scale=0.5),
    batch_tfms=aug_transforms())
dls = bears.dataloaders(path)

With the data all set, we can now use the pre-trained resnet18 model, and train it up for recognizing images of black, grizzly and polar bears.

Using Biased Data¶

As someone who is very interested in the ethics of machine learning, I want to do a quick experiment on how our model would turn out if our bears images were heavily biased. There are all kinds of real-world instances where image classifiers were trained on biased data with significant performance results. There is a well-known case identified by MIT researcher and founder of the Algorithmic Justice League, Joy Buolamwini. Her Ted Talk about it is here.

Let's see how a new model performs based on 150 images of polar bears and 15 images each of black and grizzly bears, rather than the 60/60/60 set used earlier. This is the same number of images overall, but heavily skewed to polar bears. This mirrors many biased datasets of images of humans, where the vast majority are images of white people.

In [15]:
path2 = Path('bears2')

We make directories similar to before, but with some adjustments to specify the new numbers of images:

We see that even though we gave the model the same total number of images, the biased data predictably led to a model that is less accurate with an 8.8% error rate (which is still pretty good). Let's see what that looks like in a confusion matrix:

In [24]:
interp = ClassificationInterpretation.from_learner(learn2)
interp.plot_confusion_matrix()

I was surprised to see the model correctly identify the smaller number of black and grizzly bears in the validation set. The errors actually came from mis-identifying polar bears in the set. But I suppose this makes sense. If the model struggles to identify black and grizzly bears due to reduced data, it will also have a hard time deciding if a polar bear is truly a polar bear, or one of those other types. Let's have a look at some of those errors:

In [25]:
interp.plot_top_losses(5, nrows=1)

The mis-identified polar bear images all look quite polar-bear-like to my eye, though the first one is a bit distorted, and possibly a computer-generated image. It is also important to note that the top five losses (loss is the model's metric for evaluating its predictions) are all higher than the top loss from the model we trained earlier. So the model we trained from biased data led to more errors, and less overall confidence in its predictions.

Conclusion¶

It probably is not shocking to see that biased training data negatively affects model performance. It is important to note that using balanced data does not clear all of the ethical issues with our bear classifier, either. Even if the model "performs" better, its intended use is massively important. The issue with facial recognition products is not only that they struggle with identifying underrepresented individuals. Many of their intended applications should not even be pursued in the first place. Bias in machine learning extends far beyond biased training data alone.
